### QD-1 — Independent (α, β) extraction operation

The V_quad numerical infrastructure (EXCERPTS D1, D2, D4) supplies:

- **Stokes constant S_1 ≈ 0.43770528** (8-digit), accessory parameter q_acc = (5 + i√11)/54 in Q(√−11), beta_exp = −1/(3√3).

For each candidate "what (α, β) extraction means":

**(a) Extraction via Stokes data → P_III(D_6) parameter readout:**
This is the standard isomonodromic-deformation approach: the Stokes constants S_1 and the accessory parameter q_acc determine the P_III monodromy data, which under JM-Ueno isomonodromy theory determines the P_III parameter tuple. JM 1981 Parts I–III is the canonical source for this map (Stokes data → Hamiltonian parameters). The map exists in principle and is well-defined; whether it is *operational* in the V_quad infrastructure depends on whether vquad_p3d6_recovery.py has implemented it. EXCERPT D3 records the script *names* the parameter tuple but does not say it *computes* the tuple from Stokes data — the source string "p12_journal_main.tex sec:vquad subsec Stokes data, citing Papanokechi2026Vquad" suggests the tuple was *cited* from a separate document, not extracted by the script.

**(b) Extraction via formal series matching (a_1...a_10):**
The formal series coefficients a_1...a_10 (EXCERPT D2) carry asymptotic information that, in principle, encodes the P_III(D_6) parameter tuple via the local expansion of the Painlevé transcendent at t = 0 (or t = ∞, depending on the reduction). The map "formal series → Hamiltonian parameters" is the *inverse* of the standard Painlevé asymptotic expansion. This is an algebraic/Padé-like inversion that PSLQ can attack at 200-digit precision — feasible in principle, but the inversion is non-unique without additional constraints (Stokes data fixes the branch).

**(c) Extraction via Borel transform analysis:**
rho_Borel ≈ 2.34844 carries information about the location of the nearest Borel singularity, which under resurgence relates to the leading trans-series action — but the relation to (η_∞, η_0, θ_∞, θ_0) is indirect and routes through resurgence dictionaries that are not visible in EXCERPTS D1–D5.

**Combined assessment:** the V_quad infrastructure has the *raw data* for an extraction operation, but EXCERPTS D1–D5 do not exhibit a *concretely implemented* extraction operation that produces (α_∞, α_0, β_∞, β_0) numerically from Stokes/series data without already knowing the chart-map. ### QD-2 — Numerical-fit feasibility for −1/3 offset resolution

**Discrimination between mechanisms (a), (b), (c):**

The three offset values are:
- (a) FW Prop 4.1 pull-back of h − tH = ¼v_1² − ½t to V_quad point: predicted to be −1/3 (per the round-1 §3.5.1 framing where (a) is named as a candidate origin for −1/3).

- (c) Sakai D_6^(1) root-system constant: would need to be derived from D_6^(1) simple-root data; not directly named as a numerical value in any excerpt.

Critically, (a) and (b) *both predict −1/3 as the partial sum*. They are *not numerically discriminable at the partial-sum level* — both produce −1/3.

The discriminator is the *per-coordinate distribution* of the offset:
- Mechanism (a) predicts an asymmetric distribution determined by FW eq. (4.3): the shift is ¼v_1² − ½t evaluated at V_quad's parameter point. This depends on which v_1 in FW's parametrisation maps to which project parameter, and on the value of t at the V_quad reduction. The shift is *not symmetric* across the four coordinates in general.
- Mechanism (b) predicts a *symmetric* distribution: −1/12 in each coordinate. This is the simplest fall-back precisely because it is symmetric.
- Mechanism (c) predicts whatever the Sakai D_6^(1) simple-root summation gives — not specified, but generically asymmetric.

So the numerical fit needs to extract the *full 4-tuple offset distribution*, not just the partial sum. The V_quad image (1/6, 0, 0, −1/2) shows this distribution explicitly: 1/6 in α_∞, 0 in α_0, 0 in β_∞, −1/2 in β_0. The deviation from a hypothesized "null-locus" version (where the partial sum is 0) is the offset distribution, and that distribution can be compared against (a)'s analytic prediction (after 069r3-B executes the pull-back) and against (b)'s symmetric −1/12 prediction.

**However** — and this is critical — the V_quad image (1/6, 0, 0, −1/2) is a *single point*, not a curve or family. A numerical fit at a single point cannot distinguish between mechanisms that all produce the same partial sum *unless* the distribution is itself asymmetric in a literature-anchored way. Since (1/6, 0, 0, −1/2) ≠ (−1/12, −1/12, −1/12, −1/12), mechanism (b) is *already inconsistent with the V_quad data* — the V_quad image is asymmetric, the −1/12 fall-back predicts symmetry, so (b) is in fact pre-rejected by V_quad's own image data. This is a useful observation: mechanism (b) is the simplest fall-back in §3.5.1, but the V_quad data already disfavors it.

This narrows the discrimination task to (a) vs (c). Mechanism (a) needs explicit pull-back (069r3-B's task) to produce a numerical prediction; once that prediction is in hand, 069r3-D can numerically check whether mechanism (a)'s distribution matches V_quad's (1/6, 0, 0, −1/2) image.
