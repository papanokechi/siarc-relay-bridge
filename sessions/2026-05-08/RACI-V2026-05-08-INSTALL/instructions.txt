## 4. Cadence + Escalation Tree

### Monthly cycle (Strategyzer-driven, first day of month)

1. Operator opens CMB.txt + the 4 most recent `cli_log/YYYY-Www.md`
   weekly handoffs + any new VERDICTS RECEIVED during the month.
2. Operator pastes all into a fresh Strategyzer chat with the
   standing prompt: "Produce this month's MSB and Synthesizer
   Master Prompt."
3. Strategyzer returns:
   - **Monthly Strategy Brief (MSB):** state-of-portfolio + closed-
     this-month + strategic priorities (5–10 ranked) + pipeline-
     shape questions + risk register + inputs-requested-for-next-
     MSB. Per rule6, MSB scope is portfolio + pipeline + months-
     and-quarters questions.
   - **Synthesizer Master Prompt:** a single self-contained prompt
     that Synthesizer loads on Day 1 of the month and uses as its
     standing context for all 4–5 weeks. Includes the MSB envelope
     within which Synthesizer writes its own WSBs.
4. Operator pastes the Synthesizer Master Prompt into Synthesizer
   to start the month.

### Weekly cycle (Synthesizer-driven, Sunday or Monday)

1. Synthesizer reads its standing monthly prompt + the prior week's
   `cli_log/YYYY-Www.md` + any verdicts received this week.
2. Synthesizer produces:
   - WSB within the MSB envelope (rule6: WSB owns week-level
     course corrections; cannot exceed MSB-defined portfolio scope
     without escalation).
   - Synthesizer-internal allocation of daily slots Mon–Fri.
3. Synthesizer publishes WSB to `cli_log/YYYY-Www_wsb.md`.
4. Saturday: Operator review day. Sunday: weekly handoff written.

### Daily cycle (Synthesizer-driven, every morning)

1. Synthesizer loads its standing weekly prompt + reads CMB +
   reads previous day's `cli_log/YYYY-MM-DD.md`.
2. Synthesizer writes today's relay prompt for Executer
   (≤1 page, structured per §8).
3. Operator pastes that prompt into Executer in VS Code.
4. Executer executes; outputs go to bridge repo.
5. Operator pastes Executer's terminal/result block back into
   Synthesizer.
6. Synthesizer reviews, runs any local checks (PSLQ phantom-hit
   guard, SICF if a submission is queued), arbitrates JC-class
   flags in-tier, and writes `cli_log/YYYY-MM-DD.md` summarizing
   the day.
7. End of day: Synthesizer appends today's row to the weekly log.

### End-of-week handoff (Synthesizer → next week's WSB)

Synthesizer writes `cli_log/YYYY-Www.md` summarizing the 7 days,
listing what closed, what's blocked, what should change in next
week's WSB. This file accumulates across the month; the four most
recent are the Operator's paste-input for the next monthly cycle.

### End-of-month handoff (Synthesizer → next month's MSB)

On the last working day of the month, Synthesizer writes
`cli_log/YYYY-MM_monthly_handoff.md` summarizing the month:
- 4 weekly handoffs synthesized
- Papers closed / submitted / withdrawn
- Conjectures promoted / falsified
- Milestone status changes (M1–M9)
- Methodology lessons (rule5/6 amendments queued)
- Recommended Strategyzer focus for next month

This is the document Operator pastes into the Strategyzer with
the CMB header and recent VERDICTS RECEIVED.

### Escalation tree (when Synthesizer engages Strategyzer
out-of-cycle)

Trigger out-of-cycle Strategyzer engagement ONLY for:
- **E1 (paper-level):** any P0x paper receives accept or major-
  revision verdict requiring portfolio re-prioritization.
- **E2 (pipeline-level):** a milestone (M1–M9) closes or
  forecloses; a new paper draft is proposed; a venue is to be
  added or removed from the active set.
- **E3 (methodology-level):** a rule5/rule6 violation pattern
  surfaces (e.g., a 5th synth-trust failure of a class that
  pushes the pattern from "occasional" to "structural").
- **E4 (calibration):** SICF returns HOLD or detects a
  calibration failure pattern in `critic_calibration_log.txt`.

NOT escalations (handle in Synthesizer tier):
- Single-experiment falsifications or confirmations.
- Routine HALT codes during execution.
- JC-class judgment calls during execution.
- Week-level course corrections within MSB envelope.
- SICF MINOR-REVISION verdicts.
- Single blocker arbitrations.

## 8. Relay Prompt Protocol

All Executer tasks are issued as structured relay prompts with:
- TASK name and AEAL class
- BRIDGE path for git commit
- BACKGROUND context
- Numbered STEPS
- HALT IF conditions
- Explicit: "Do not commit or push" or commit instructions

Under the new RACI, the *author* of relay prompts is Synthesizer
on all daily and weekly-scope tasks. Strategyzer authors
prompts only if a Strategyzer-altitude artifact is being drafted
(e.g., a new paper kickoff prompt, a major announcement draft,
or a methodology amendment).

**Phantom hit prevention (mandatory in all PSLQ sessions):**
Any PSLQ relation with effective L-coefficient = 0 is REJECTED.
Effective L-coefficient is interpreted per W19 audit as the sum
of L-bearing basis-element coefficients (L, L·π, L·π², L·log2,
etc.), not the L-only column. This prevents the ζ(2)=π²/6 and
φ=(1+√5)/2 traps without over-rejecting genuine relations where
L appears in product form (e.g., 12 − L·π = 0).
Non-negotiable; appears in every PSLQ relay prompt.

## 15. SICF (Structured Internal Criticism and Framing)

Before submitting any paper, run a four-agent SICF review:
  Agent 1 — Advocate: strongest case for acceptance
  Agent 2 — Referee Critic: most likely rejection reasons
  Agent 3 — Framing Advisor: venue and presentation
  Agent 4 — Next Steps Planner: what to do after SICF

Under the new RACI, SICF is invoked by **Synthesizer** during
the week using the API-FREE prompt template + a fresh Claude.ai
tab for each role (per `docs\SICF_API_FREE_prompt.md`). The
Strategyzer reviews SICF outputs in monthly synthesis pass,
specifically for:
  - Patterns in calibration failures
    (`critic_calibration_log.txt`)
  - Patterns in HOLD verdicts (suggesting a paper-class
    methodology gap)
  - Patterns in venue mismatch (suggesting portfolio
    re-shaping)

Per E4 escalation, Synthesizer surfaces calibration failures to
Strategyzer out-of-cycle. SICF MINOR-REVISION verdicts stay in
Synthesizer tier.

SICF verdict determines: PUBLISH-READY / MINOR-REVISION /
MAJOR-REVISION / HOLD / WITHDRAW-AND-RESUBMIT before any
submission.
