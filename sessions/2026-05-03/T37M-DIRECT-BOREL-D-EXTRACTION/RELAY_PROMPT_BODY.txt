0. CONTEXT (READ THIS FIRST)
================================================================

The d=2 PCF Birkhoff transseries has the structure:
    a_n ~ C * Gamma(n) * zeta_*^(-n) * (1 + a_1/n + a_2/n^2 +...)
          + D * Gamma(n) * (2*zeta_*)^(-n) * (1 + ...)
          + E * Gamma(n) * (3*zeta_*)^(-n) * (1 + ...)
          + ...

The Stage-2 LSQ approach (017c, 017e) attempts to:
  1. fit C and {a_k} from large-n data,
  2. subtract the leading term,
  3. fit D from the residual.

This struggles because the leading-term subtraction must be
exact to ~50 digits to leave a clean D signal at the next
rung. With cached / extended series at dps=400 N=5000, the
fit residual is dominated by polynomial-in-1/n corrections
to the LEADING amplitude, which Stage-2 cannot disentangle
from the next-rung amplitude — and pushing precision to
dps=400 / 800 / ... only postpones the problem (rel half-
range went from 10^4 at dps=250 to 10^217 at dps=400; doesn't
shrink with precision because the Stage-2 conditioning fails
algebraically).

The DIRECT BOREL approach reads D off without projection:

    B[a_n](xi) := sum_{n>=0} a_n * xi^n / Gamma(n+1)

For a typical Birkhoff series, B[a_n] is meromorphic in a
disk around xi = 0 with the nearest singularities on the
positive real axis at:
    xi_1 = zeta_*       (leading; residue ~ C * f_1(D, ...))
    xi_2 = 2 * zeta_*   (next rung; residue ~ D * (sign factor))
    xi_3 = 3 * zeta_*   (next-next; residue ~ E)
    ...

Padé approximants of B[a_n] capture these singularities
explicitly: an order-[M, M] Padé has up to M poles, and the
nearest 3-5 poles approximate xi_1, xi_2, xi_3, ... with
residues that converge as M increases.

The methodology is well-established in resurgence / borel
summation (Costin "Asymptotics and Borel Summability";
Loday-Richaud "Divergent series, summability and resurgence";
the M.Marino-Reis "Resurgence in Quantum Field Theory" review).
What 017m brings is the AEAL discipline: cross-validation
across multiple Padé orders, multiple windowed truncations,
hard hygiene against spurious poles, and PARTITION TEST on
|D| / |S_2| against Delta_b sign.

Goal (one sentence): extract D / S_2 per rep via direct
Borel-Padé singularity analysis at xi = 2*zeta_*, with
cross-validated AEAL bounds, and test the |S_2| Delta_b
partition that Stage-2 LSQ could not measure.

PHASE A — INPUT VALIDATION + BOREL TRANSFORM SETUP
---------------------------------------------------

A.1  Load 017e per-rep series a_n for n in [0, N_max],
     N_max = 5000 (or whatever 017e cached). Verify
     load_count = N_max + 1 per rep; halt
     T37M_INPUT_TRUNCATED otherwise.

A.2  Load (alpha, beta, gamma, A, Delta_b, zeta_*) per rep
     from T35 representatives.json. Compute target
     singularity locations xi_1 = zeta_*, xi_2 = 2*zeta_*,
     xi_3 = 3*zeta_*.

A.3  POLYNOMIAL-CORRECTION SUBTRACTION (cleanness step):
     The Borel transform B[a_n](xi) of the LEADING amplitude
     C * Gamma(n) * zeta_*^(-n) * (1 + a_1/n + a_2/n^2 + ...)
     is meromorphic with leading pole at xi = zeta_*. To
     improve Padé conditioning at xi = 2*zeta_*, OPTIONALLY
     subtract the polynomial-correction-RESUMMED leading
     transmonomial using 017e's measured a_1, ..., a_K (K=25)
     before computing the Borel transform. This CLEANNESS
     STEP makes xi_2 the dominant singularity in the
     residual series.

     Concretely: define
         a_n_residual := a_n - C * Gamma(n) * zeta_*^(-n) *
                                sum_{k=0}^{K} a_k / n^k
     (with a_0 := 1) and apply Borel-Padé to a_n_residual
     instead of a_n.

     If 017e's K_lead=25 a_k values are not all available
     to >= 60 digits per rep, halt T37M_LEADING_DATA_INCOMPLETE.

     Cross-check: a_n_residual should decay as
     |D| * Gamma(n) * (2*zeta_*)^(-n) at large n (factor
     ~2^n smaller than a_n itself). At n = 5000, residual
     should be ~10^-1500 below a_n in absolute value (test
     to within 10x).

A.4  Record borel_transform_per_rep.json with per-rep
     (a_n_residual_first_5, a_n_residual_last_5,
     leading_subtraction_mode "subtracted" / "raw",
     residual_decay_estimate, ...) for cross-validation.

PHASE B — BOREL TRANSFORM + PADÉ APPROXIMANTS
----------------------------------------------

B.1  Per rep, compute the Borel transform coefficients
     b_n := a_n_residual / Gamma(n+1)
     for n in [0, N_max]. Use mpmath at dps=400.

B.2  Compute Padé approximants P_M(xi) of order [M, M] of
     the truncated Borel polynomial sum_{n=0}^{M_in} b_n * xi^n
     for:
         M_in in {2000, 3000, 4000, 4900}
         M    in {N_in/3 - 50, N_in/3, N_in/3 + 50}
     (i.e., each M_in gives 3 Padé orders; total 12 Padé
     approximants per rep).

     Use mpmath.pade or sympy.series.formal.pade. If the
     library does not provide Padé at mpmath precision,
     implement directly via the linear-algebra recipe
     (solve M Hankel-like equations for numerator + denominator
     coefficients).

B.3  Per Padé approximant P_M, locate POLES in the complex
     xi-plane via mpmath.findroot or mpmath.polyroots on the
     denominator. Filter to physical poles:
         |xi - real_axis| / |xi| < 0.05  (essentially real)
         Re(xi) > 0                     (positive real axis)
         |xi| < 5 * |zeta_*|            (within first 5 rungs)

B.4  Per Padé, identify the pole CLOSEST to xi_2 = 2*zeta_*
     (within tolerance 0.1 * |zeta_*|). Record:
         xi_2_pade (complex value),
         |xi_2_pade - 2*zeta_*| / |2*zeta_*|,
         residue_at_xi_2_pade (computed via
                                lim_{xi -> xi_2_pade}
                                (xi - xi_2_pade) * P_M(xi)).

B.5  Halt T37M_PADE_DIVERGENT if:
       - All 12 Padés (per rep) fail to find a pole near
         xi_2;
       - OR the spread of |xi_2_pade - 2*zeta_*| across
         the 12 Padés is > 1% (i.e., locations don't
         converge);
       - OR the spread of residues across the 12 Padés is
         > 10% (i.e., residue value doesn't converge).

B.6  Halt T37M_BOREL_SINGULARITY_NOT_AT_2ZETA if a clearly-
     defined pole is found at a location significantly
     offset from 2*zeta_* (i.e., the Borel-plane structure
     is not what the leading-order Birkhoff theory predicts
     — methodology question, not a normal failure).

PHASE C — D EXTRACTION + S_2 PARTITION
---------------------------------------

C.1  Per rep, take the median residue from the 12 Padé
     approximants at xi_2_pade as the canonical estimate:
         D_estimate = median(residue_at_xi_2_pade) / convention_factor
     where convention_factor accounts for the Birkhoff-
     transseries convention used by T35 / 017c (read this
     factor from T35's documentation; it is typically
     (2*pi*i) or 1 depending on whether D is defined as
     S_2 / (2*pi*i) or as S_2 itself).

C.2  Compute envelope half_range_relative as
         (max(residues) - min(residues)) / (2 * |median|)
     across the 12 Padés per rep.

C.3  S_2_per_rep := 2 * pi * i * D_estimate (if convention_factor
                    chose D := S_2 / (2*pi*i)) or = D_estimate
                    (otherwise). Document.

C.4  PARTITION TEST: do |S_2| values partition by Delta_b sign?
     With T35's reps grouped as
         Delta_b < 0: V_quad (-11), QL15 (-20)
         Delta_b > 0: QL05 (+8), QL09 (+1)
     compute the gap
         gap := min{|S_2_neg|} - max{|S_2_pos|}
     across the 4 reps' D_estimate values.

     If gap > 5 * (max envelope half_range across reps),
     ORDERING_PASS. Otherwise INDETERMINATE.

C.5  CROSS-VALIDATION against 017c's d_extraction_feasibility:
     for each rep, the 017m D_estimate should be:
       - finite, non-zero,
       - magnitude consistent with 017c's infeasible-Stage-2
         estimates (within 10x of any 017c attempt's median;
         this is a very loose check, since Stage-2 was wildly
         unstable);
       - sign-stable across all 12 Padés (for clean
         extraction).

C.6  Record d_extraction_per_rep.json + s_2_per_rep.json
     with per-rep (D_estimate, |D|, arg(D), envelope,
     ordering_pass, padé_orders_used, etc.).
